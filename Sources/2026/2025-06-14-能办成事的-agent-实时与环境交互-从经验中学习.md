# [2025-06-14] 能办成事的Agent：实时与环境交互，从经验中学习

## TL;DR

Conference talk by 李博杰 (co-founder/chief scientist of [[Pine AI]]) at A2M Summit, covering two core challenges for real-world agents: high latency in real-time interaction and inability to learn from experience. The talk presents fast-slow thinking, code-generated RPA tools, knowledge bases, and SFT/RL as the four pillars of a production agent system.

## Key claims

- Two fundamental gaps in current agents: (1) no real-time bidirectional interaction — agents batch-process tasks without mid-task human exchange; (2) no experience accumulation — every run is i.i.d., resets state, and cannot improve from prior attempts. These gaps mirror OpenAI researcher 姚顺雨's essay *The Second Half*.
- **Fast-slow thinking**: a "Talker" agent handles real-time interaction (voice calls, GUI); a "Reasoner" agent runs in the background doing multi-step planning and stores decisions in shared memory. Talker is trained via SFT+RL to stall on critical decisions ("can you tell me more?") while waiting for Reasoner's verdict.
- **Code-generated RPA tools**: LLM generates deterministic RPA scripts for fixed UI sequences; these are split into atomic sub-tools at decision boundaries where LLM judgment is required. Result: GUI task completion drops from 9 model calls / 47 seconds to 2 model calls + 1 RPA tool call for Google search; airline booking drops from 19 minutes (240 API calls) to 4 minutes on repeat runs — 5× faster than [[Anthropic]] computer use.
- **RL role**: RL cannot exceed the base model's capability ceiling — it can only reinforce outputs the model has already produced. Its value is raising success rate from 10% to 90% (熟练度, skill), not raising intelligence.
- **Tau-bench evaluation**: on the Airline scenario, Claude 3.7 Sonnet, Claude 4 Sonnet, and Claude 4 Opus all score ~60% success. Pine's sequential-revision method (a "judge" model approves/rejects each irreversible action) raised the baseline from 56% to 64%. Of 18 failures in 50 Airline test cases, 8 had incorrect ground truth (GT violates the framework's own rules or contains factual errors), 4 more were solvable with creative approaches the annotators missed — leaving only 6 genuine agent errors.
- **LLM-as-Judge** is recommended when rules are clear and context is complete; weak models will approve everything and miss errors; SOTA models with thinking mode are required.
- Three tiers of world interaction: voice phone calls (~15–50 tokens/sec input), GUI operations (~2000 tokens/screenshot), physical world (robots/autonomous driving, up to 20K tokens/sec); each tier demands faster reaction and higher reliability.
