# [2025-03-23] 从零开始的veRL框架解析

## TL;DR

A thorough bottom-up technical walkthrough of the [[verl]] RLHF framework for algorithm engineers, covering parallelism fundamentals (SPMD, ZeRO, TP, PP, EP), [[Ray]] actor model, Hybrid Engine design, and practical hyperparameter rationale — written by a former insider as an exit-from-industry tutorial.

## Key claims

- [[verl]] uses [[Ray]] Actor abstraction to orchestrate multiple model modules (Actor, Rollout, Ref, Critic, Reward) with independent resource allocation and task scheduling, overcoming the limitations of single-script multi-process setups (deepspeed/torchrun) for multi-model RL workflows.
- ZeRO stages: ZeRO-1 shards optimizer states (1/N), ZeRO-2 adds gradient sharding, ZeRO-3 shards weights too but increases communication 1.5× — ZeRO-1/2 leave forward unaffected; ZeRO-3 requires all-gather before each forward/backward.
- SPMD (Single Program Multiple Data) is the foundation of DDP/DeepSpeed/Megatron: all ranks run identical code, differentiated only by rank environment variables; no central controller exists at runtime, making it efficient but mentally heavy to write.
- Parallelism taxonomy: Data/Sequence Parallel = split input X (simple); Tensor/Pipeline/Expert Parallel = split weights W (complex). ZeRO-3 is still data-parallel — computes on complete params after all-gather, unlike TP/PP which compute on partitioned params.
- Hybrid Engine (verl's key feature): single set of GPUs switches between training (ZeRO/FSDP) and generation (vLLM TP) modes without copying weights between separate GPU pools, trading resharding overhead for memory efficiency.
