# [2025-09-01] torch.compile 训练的现状总结（2025年8月）

## TL;DR

PyTorch 编译器团队核心成员 Edward Yang 的官方博客（由代老板中文转译并加注），系统梳理截至 2025 年 8 月 `torch.compile` 在大规模训练领域的现状：DTensor、先进并行化路线图、优化器（Inductor/CUDA Graph/AC/FP8/Flex Attention）、编译时间改善和如何上手。

## Key claims

- `torch.compile` 较 Eager 模式通常带来 1.5–2× 加速，并使自动激活检查点（automatic activation checkpointing）和异步张量并行（async tensor parallelism）成为可能。
- [[DTensor]] 是 PyTorch 分片张量的标准抽象（SPMD device mesh 导向），已与 FSDP2、SimpleFSDP、AutoParallel、[[Torchtitan]] 等集成；[[FSDP2]] 用 DTensor 取代 FSDP1 的不透明 `FlatParameter`，PyTorch 官方已弃用 FSDP1。
- `torch.compile` 关键非功能性特性：JIT（首次调用时阻塞编译）、与 Eager 可组合、梯度更新延迟至编译区域末尾、图中断（graph break）透明绕过、函数调用默认内联（编译时间与 Transformer block 数量成正比，可用 regional compilation 优化）。
- Inductor 为默认后端，生成 Triton kernel，自动调优 cuBLAS/CUTLASS/Triton matmul 实现；CUDA Graph 集成提供比手动应用更好的健全性保证；Flex Attention 下游仓库用户已达 632 个（较 2025-01 的 125 个大幅增长）。
- 大规模训练未来愿景：SimpleFSDP 按 FSDP 模式插入集合操作再用 pass 优化；AutoParallel 设定 GSPMD 风格目标，自动发现数据并行、张量并行、专家并行分片方案；当前路线 fork [[Torchtitan]] 作为训练栈基础。
- PyTorch 与 JAX 的分叉路径对比：JAX 从 XLA 核心出发（编译驱动并行领先数年，但需越来越多手动 escape hatch）；PyTorch 从完全手动分布式模式出发，正逐步添加自动化机制。
