# [2026-03-24] Composer 2 Technical Report

## TL;DR

Composer 2 is Cursor's domain-specialized agentic coding model, trained via continued pretraining on Kimi K2.5 (1.04T params, 32B active MoE) followed by large-scale asynchronous RL. It achieves 61.3 on CursorBench-3, 73.7 on SWE-bench Multilingual, and 61.7 on Terminal-Bench — competitive with GPT-5 class models at significantly lower inference cost.

## Key claims

- Base model: Kimi K2.5 (1.04T / 32B active MoE); continued pretraining in three phases — bulk at 32k context, long-context extension to 256k, short SFT — performed in MXFP8 on NVIDIA B300s using AdamW. Lower pretraining loss is a reliable predictor of downstream RL reward, validated on Qwen3-Coder-30B-A3B.
- RL uses a policy gradient with multiple samples per prompt (GRPO-style), single-epoch regime, fully asynchronous training/rollout workers via Ray; removes length standardization from GRPO and avoids normalizing group advantages by std, finding both cause degeneracy.
- KL regularization: uses k1 = -log r estimator instead of the common k3 = (r-1) - log r because k3 variance blows up as p and q diverge; both average reward and best-of-K improve together during training, contradicting the "RL merely reweights known paths" narrative.
- Self-summarization chains multiple generations with summaries within one rollout using the final reward on all tokens; outperforms separate prompt-based compaction while using fewer tokens and reusing KV cache; model learns to summarize multiple times on hard examples.
- CursorBench-3: tasks have median 181 lines changed (vs. 7-10 for SWE-bench Verified) and median 390-char prompts (vs. 1,185-3,055 chars for public benchmarks); continuously refreshed from real Cursor engineering team sessions to avoid contamination and saturation.
- Composer 2 vs. Composer 1: 61.3 vs. 38.0 on CursorBench (61% improvement); vs. Kimi K2.5 base (36.0), demonstrating the value of the continued pretraining + RL stack.
- Weight sync for RL inference: delta compression transmits only diff from last upload; even for 1T-parameter model, diffs compress to a few GB per step; sharded upload/download across training ranks and inference replicas; geo-distributed inference across US and Europe clusters over commodity S3.
- MoE precision: NVFP4 (with FP8 per-block scales and FP32 per-token scales) for forward pass on NVIDIA Blackwell to match inference; MXFP8 for backward pass; IEEE-compliant float ops required for NVFP4 (fast-approximation causes divergence after ~100 RL steps).
- Context Parallelism (CP) replaces TP as primary long-context axis — less communication, preserves full hidden dimensions; EP decoupled from TP enabling larger expert-parallel degrees; DeepEP used for token dispatch/combine; EP=8, CP=2 for pretraining; EP=8, CP=8 for RL.

## What this changes

Establishes that domain specialization via continued pretraining + RL can match frontier general-purpose models in software engineering at lower cost; validates CursorBench as a contamination-resistant alternative to SWE-bench; demonstrates self-summarization as a practical technique for long-horizon agent coherence.
