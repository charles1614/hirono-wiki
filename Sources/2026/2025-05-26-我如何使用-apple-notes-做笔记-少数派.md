# [2025-02-24] 我如何使用 Apple Notes 做笔记

## TL;DR

Personal workflow essay describing two years of using [[Apple Notes]] with a modified [[PARA Method]] for all note-taking. Argues that reducing friction and serving action (not aesthetics) are the highest-priority criteria for a note system, and that features like backlinks, tags, and rich formatting are non-essential for most users.

## Key claims

- [[Apple Notes]] chosen for: system-native (no install), iCloud sync without account registration, unlimited attachment support (file/image/audio), and fast launch — all reducing capture friction.
- Implemented PARA (Projects/Areas/Resources/Archive) with two additions: a **Drafts** folder for unstructured mobile captures and an **Inbox** folder for unvetted project ideas before formal commitment.
- Author's primary workflow: create a Project folder for each ongoing piece of work; any related information encountered goes directly there — eliminates the "where to put this note" friction permanently.
- "The most important thing is to record what is currently relevant to your actions" — backlinks and double-links go unused without measurable information loss over two years of practice.
- Migration from old tools not required: start fresh in [[Apple Notes]], find old data only if actually needed.
