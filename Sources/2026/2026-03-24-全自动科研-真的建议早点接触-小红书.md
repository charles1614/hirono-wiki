# [2026-03-24] 全自动科研，真的建议早点接触

## TL;DR

Practitioner account of running Karpathy's [[AutoResearch]] for one week on small-model training optimization, with a curated list of community-derived variants; key insight: the evaluation criterion must be defined correctly before automation, or the agent optimizes in the wrong direction.

## Key claims

- Core AutoResearch loop: give AI a goal + evaluation criterion → let it modify, test, and retain improvements autonomously; suitable for overnight batch runs of hundreds of iterations.
- Critical caveat: "评判标准得想清楚。标准不对，AI 优化方向就偏了" — the author burned the first night because the reward signal was ill-specified.
- Community variants catalogued: Claude Autoresearch (github.com/uditgoenka/autoresearch, lowest barrier), Codex Autoresearch (coding performance optimization), AutoResearchClaw (idea-to-paper pipeline), autoresearch-agents (LangChain founder, github.com/hwchase17/autoresearch-agents), AutoKernel (auto CUDA kernel optimization, github.com/RightNow-AI/autokernel).
- Canonical project: github.com/karpathy/autoresearch.
