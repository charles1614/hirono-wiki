## [2026-05-12] refactor | Active-tier Entity Observations backfill + reindex date-format fix

Two changes that close the Karpathy-compounding gap for the 9 active-tier
entities + fix a long-standing frontmatter-mutation bug.

**Part A — Active-tier Observations backfill.** Every entity with refs ≥ 3
(active tier) now has cited Observations populated from the 16 ingested
Sources, with a multi-sentence Synthesis paragraph at the top. The 9
entities: [[NVIDIA]] (6 refs), [[Blackwell]] / [[Hopper]] / [[SGLang]] /

[[TensorRT-LLM]] (3 refs each). Total: ~30 cited Observations + 9

Discipline: each Observation is one atomic claim, cited with
`[[Sources/<slug>]]`. Sourced from the Source's TL;DR + Key claims; not
re-reading the raw. The Synthesis paragraph is a 3-5 sentence weave of
the cross-source picture, regenerable from the Observations below it.

Result: clicking any active-tier entity wikilink now reaches a page that
tells you (a) what the thing is, (b) what every Source that cited it
said about it, with citations. The dead-end-stub failure mode is closed
for the load-bearing entities.

Seen-tier entities (refs ≤ 2) remain scaffolded; they'll accumulate
observations naturally as more Sources land. Reindex output shows the
"missing observations" list now starts at refs=2.

**Part B — reindex.ts date-format mutation bug (D7).** Long-standing
issue caused by `matter.stringify`: YAML parses unquoted dates
(`2026-05-12`) as JavaScript `Date` objects, then re-serializes them
as full ISO timestamps (`2026-05-12T00:00:00.000Z`). reindex.ts was
silently mutating the on-disk date format on every entity/topic
rewrite, drifting from the YYYY-MM-DD schema convention.

Fix: `normalizeDateFields()` helper in reindex.ts coerces Date /
ISO-stringified-Date values back to YYYY-MM-DD before
`matter.stringify`. Applied at both `planEntityUpdate` and
`planTopicUpdate` write paths. One-time sweep across 185 files
restored the convention; reindex now preserves it.

Verification:
- Tests: 1103/1103 pass.
- Lint: 0 errors, 40 warnings (orphan seen-tier stubs awaiting their
  first incoming reference), 348 info.
- `grep -rl "T00:00:00" Entities Topics Sources Meta` → 0 files.
- Reindex run is idempotent: re-running doesn't mutate any file's
  date frontmatter.

## [2026-05-11] refactor | Pre-scale Sources schema lockdown

Before scaling from 16 ingested Sources to 200+, locked in the Sources page
schema in `Meta/schema.md`. Seven decisions:

1. **Per-Source `## Open questions` → DROPPED.** Per-Source questions don't
   compound (locked to a page future-you won't reopen). Karpathy's gist
   treats unresolved questions as a lint-time cross-cutting concern, not a
   per-page section. Cross-source research questions now go in
   `Topics/<X>.md ## Open threads`; source-specific re-fetch TODOs go in
   the `## Raw source` footer; source-internal critique folds into the
   relevant `## Key claims` bullet.
2. **`## What this changes` → documented as OPTIONAL** (was project-extension,
   not in schema). Substantive papers / announcements keep it; light blog
   posts and small PRs that would just paraphrase Key claims skip it.
3. **`## Visual observations` → kept optional** with the existing heuristic.
4. **`## Troubleshooting essentials` → not formalized.** Single instance
   (TensorRT-LLM) folded into Key claims as a tail bullet.
5. **`tags:` frontmatter → lint-required.** Sources without ≥1 tag now fail
   lint (`tools/bin/lint.ts` change).
6. **`highlights:` frontmatter → dropped from schema spec** (verified
   zero usage across all Sources + raw source.json files).
7. **Section ordering → documented, not lint-enforced.**

Sweep of the 16 existing Sources/2026/* pages:
- All 16 had `## Open questions` removed.
- 40 cross-source research questions migrated to `## Open threads` on 16
  Topic pages (most heavily populated: Distributed-Serving Observability,
  Low-Precision Training, MoE Serving, Attention Kernels, Communication
  Overlap).
- 2 Sources lost their `## What this changes` section (nano-vllm, CUDA 13.1
  — both were paraphrase-of-Key-claims).
- 1 Source lost its `## Troubleshooting essentials` section (TensorRT-LLM
  — gotchas folded into Key claims).
- 11 Sources had `Pairs with: [[X]]` bullets dropped from `## What this
  changes` (cross-links already exist via `## Entities touched` /
  `## Topics touched`).

Verification: 1097/1097 tests pass (new tags-required-list test added).
Lint went from 96 → 93 errors (3 dead-wikilinks inside dropped Open
questions sections were removed). Sources index unchanged (16 entries).
Reindex picked up Topic-page changes; no new orphans.

End state: schema documents exactly the sections that compound across the
corpus; lint enforces frontmatter + wikilink + image-orphan rules but
leaves prose discipline to convention. Future ingests have a clean
template to follow before the 200-page batch.
