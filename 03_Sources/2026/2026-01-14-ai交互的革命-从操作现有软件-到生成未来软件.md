# [2025-11-26] AI交互的革命：从操作现有软件，到生成未来软件

## TL;DR

系统梳理AI Agent交互范式的三重演进：API Agent（调接口）→ [[GUI Agents|GUI Agent]]（模拟点击）→ Generative UI（现场生成软件），以[[Li Auto]]理想同学MCP+CUA混合架构为落地案例，以Google GenUI为未来范式。

## Key claims

- **API Agent vs GUI Agent分歧**：API-First模式高效精准但受限于开放接口；GUI Agent通过多模态模型截图识别按钮坐标，实现"只要屏幕能显示就能操作"的通用性，代价是速度慢且对软件改版脆弱。
- **理想同学双脑架构**：MCP（API Agent）负责车辆硬件控制（毫秒级响应、绝对安全），CUA（Cockpit Using Agent，GUI Agent）负责第三方App操作；以星巴克点咖啡为例，CUA通过截屏→识别→点击→多步交互完成无API接口的App操控。CUA的ScreenSpot能力达到72.7%（来自Gemini 3 pro基准）。
- **Generative UI（GenUI）**：Google研究《Generative UI: LLMs are Effective UI Generators》展示AI直接用Agentic Coding生成用户专属交互界面，从"操作现有软件"跃升为"现场造软件"；案例包括根据查询生成交互式分形探索器，以及千人千面的自适应UI。
- 三重境界总结：API时代AI是连接器，GUI时代AI是模仿者，GenUI时代AI是重构者，实现从"人适应软件"到"软件适应人"的反转。
