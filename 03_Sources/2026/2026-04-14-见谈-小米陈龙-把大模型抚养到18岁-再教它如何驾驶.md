# [2026-04-13] 见谈｜小米陈龙：把大模型抚养到18岁，再教它如何驾驶

## TL;DR

A ~32 KB weixin technical interview by 一见Auto with **Chen Long (陈龙)** — head of Xiaomi's smart-driving foundation-model team (formerly at Cambridge-incubated **Wayve**, an early VLA-for-driving pioneer). Chen Long lays out Xiaomi's evolution from **end-to-end + VLM** → **VLA** → **XLA** (eXtended VLA: adds sound, radar, robot-data modalities beyond V/L/A) and the **two-stage training philosophy**: *"raise the model to age 18"* (general multimodal + spatial + physical-commonsense + robot data) → *then teach driving* via **Latent CoT** (high-dim machine-language reasoning in latent space, vs verbose human-language CoT). The Latent CoT step is the load-bearing engineering: on-vehicle compute can't afford full natural-language chain-of-thought, so Xiaomi trains the model to "think silently in machine code" while still being externally interpretable via paired-decoding (Latent → human-language CoT + future-video prediction). Lei Jun committed **≥¥60 B over 3 years** to model + embodied + AI-application R&D across the "人车家" (people-car-home) ecosystem.

## Key claims

**XLA = VLA + extra modalities** (per Lei Jun's SU7 launch, March 2026): the "X" is **multimodal cognitive input** — visual + radar + nav + **sound** + **robot data**. Differentiator vs vanilla VLA is the latter two: the model gets driving-adjacent embodied perception. Xiaomi's framing: a model that has lived in the physical world (via robot training) drives better.

**Two-stage cognitive training** (Chen Long's "raise to 18 then learn driving" framing):

- **Stage 1 (pre-Nov 2025): inherit MiMo-VL base.** Luo Fuli (罗福莉) joined Xiaomi; her team open-sourced **Xiaomi MiMo-VL** + shared training framework. Chen Long's team built on this rather than from scratch.
- **Data-order matters**: initial attempt to mix all data (general + driving + robot) simultaneously DEGRADED both driving and robot performance. Fixed by sequencing: **general multimodal + spatial commonsense → then driving + robot data** — the "first become a person, then drive" analogy.
- **CoT data labeling** as the team's core technical innovation: every driving scenario gets a structured chain-of-thought annotation (identify objects → understand object states → decide action). Bridges the driving + robot data fusion problem.
- **Stage 2 (Nov 2025+): MiMo-Embodied released open-source** (2025-11-21). Modal capability: autonomous thinking. But latency problem: full natural-language CoT is too slow for on-vehicle inference.

**Latent CoT** = high-dim machine-language reasoning that runs silently inside the model:

- **Three thinking modes injected**: (1) intuition / end-to-end, (2) language-based for sign/navigation reading, (3) imagination-based future-scene prediction (world-model-style).
- **Architectural trick to keep it interpretable**: train the model to *paired-decode* the latent state into BOTH a human-language CoT trace AND a predicted-future-1-second video. Reward = both decodings agree with ground-truth; Loss = either one diverges. This is what the training-loop flowchart visualizes.

**World-model taxonomy** Chen Long distinguishes:

- **World Simulator Model** — generates infinite driving scenarios for in-sim training.
- **World Action Model** — predicts future driving frames directly, drives decision-making at inference time.

The implication for the industry debate (VLA vs world-model): Chen Long argues they're **complementary, not exclusive** — VLA provides language-grounded interpretability; world model provides spatial-temporal imagination. XLA fuses both.

**Wayve provenance**: Chen Long previously at Wayve (Cambridge UK autonomous-driving). Wayve's known technique cited: built a "driving commentary" dataset by having human drivers **speak aloud their decisions while driving** — providing CoT-style supervision pairs.

**Scaling Law in VLA era**: classical Scaling Law's marginal returns are diminishing in VLA; Chen Long says a new Scaling-Law curve requires *new-modality* (V+L+A jointly) data. "One question, one image, one answer" is only one data unit in VLA — needs re-labeling at scale via Luo Fuli's AI-infra + MiMo-VL pipeline (machine-generated + human-screened).

**Compute / deployment constraint**: even after model compression, on-vehicle compute can't run full natural-language CoT for real-time driving. This is the load-bearing latency constraint that justifies Latent CoT specifically.

## What this changes

A second-source confirmation of two trends already in the corpus:

- **MiMo-VL / Luo Fuli's team is the basis layer for multiple Xiaomi efforts**, not just inference research. Cross-ref to [[2026-04-27-deepseek-v4砍掉mla-一个月前有人预言了-小红书]] — Luo Fuli's V4-prediction interview was an outside-Xiaomi view; this Source is an inside-Xiaomi view of her team's role. The shared training framework + open-source MiMo-VL is reusable infra for any team building specialized vertical models.
- **Latent CoT as a serving-economics solution to CoT-inference-latency**: parallel to [[Speculative Decoding]] and KV-cache-reduction techniques (V4's CSA + on-disk KV) — all three are different mechanisms for "deliver good-quality reasoning at production latency." Latent CoT is the embodied-AI variant. Worth tracking whether the same trick (latent reasoning + paired interpretability decoder) crosses back into pure-LLM stacks.

**Open thread**: the Wayve-style human-narrated driving-commentary dataset (humans speaking decisions aloud while driving) is a CoT-supervision-at-scale recipe that doesn't appear elsewhere in the corpus. If self-driving teams are systematically generating CoT-grounded behavioral data, that's a corpus the wider LLM community could learn from.

The interview frames embodied AI / autonomous driving as **adjacent-but-distinct from the LLM-systems theme** the wiki has so far. Chen Long's "raise to 18 then drive" is a transfer-learning argument; the Latent CoT mechanism is the load-bearing kernel.
