# [2024-07-29] ByteCheckpoint: A Unified Checkpointing System for Large Foundation Model Development

## TL;DR

ByteCheckpoint is ByteDance's industrial-grade checkpointing system for large-scale LFM training, accepted at NSDI'25. It introduces parallelism-agnostic checkpoint representation enabling automatic load-time resharding, supports Megatron-LM/FSDP/DDP/veScale frameworks and multiple storage backends, and achieves up to 54.20× reduction in checkpoint stalls vs open-source baselines.

## Key claims

- Checkpoint resharding is routinely demanded in production: the authors recorded 1,870 pre-training resumption reshards, 13,080 cross-stage transitions, and 19,844 evaluation reshards over six months on their AI platform, with average completion times of 1,870s / 650s / 593s for the three scenarios when done via offline scripts.
- ShardMeta abstraction: each tensor shard is represented as (fqn, nD_offsets, nD_lengths), decoupling the stored format from runtime parallelism; a global metadata file consolidates all shard metadata, enabling any new parallelism configuration to reconstruct the complete tensor at load time without re-running an offline job.
- Irregular tensor decomposition: ZeRO-sharded tensors (Megatron ZeRO2, FSDP ZeRO3) that cannot be expressed as n-dimensional slices are decomposed into multiple regular ShardMeta entries; this avoids costly all-gather during save while preserving reshardability at load time.
- Fully asynchronous saving pipeline with balanced zero-redundancy plan generation and efficient irregular-tensor processing; scales to 405B LFM training on 8,960 GPUs.
- Compared to PyTorch DCP and Megatron Distributed Checkpoint: 12.13–161.50× improvement in checkpoint stall reduction; average 6.05× faster saving and 3.88× faster loading.
- Dataloader states are split into replicated (saved once by rank 0) and sharded (saved per I/O worker), simplifying resharding of dataloader state when parallelism changes across stages.
- On the platform, top three frameworks are DDP (25,393 jobs, avg 6 GPUs), FSDP (16,842 jobs, avg 25 GPUs), Megatron-LM (13,727 jobs, avg 301 GPUs); ByteCheckpoint provides a single API layer above all three.

## What this changes

Eliminates manual resharding scripts as the dominant operational cost in LFM development; makes load-time resharding the default rather than an offline engineering task; enables training resumption and evaluation to proceed immediately without checkpoint format conversion.
