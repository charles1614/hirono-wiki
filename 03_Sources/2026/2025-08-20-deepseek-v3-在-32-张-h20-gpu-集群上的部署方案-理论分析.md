# [2025-08-20] DeepSeek-V3 在 32 张 H20 GPU 集群上的部署方案【理论分析篇】

## TL;DR

An earlier theoretical analysis (pre-Tencent benchmark update) of deploying [[DeepSeek-V3]] on 32×[[H20]] GPUs with [[vLLM]]. Reaches a more optimistic conclusion — all four SLO targets (50,000 tokens/s, 200 concurrency, 32K context, TTFT < 0.8 s) deemed achievable through optimization — compared to the updated version which revises this down significantly after real-world data.

## Key claims

- Recommended config is identical: EP=32, TP=1, PP=1 with 8 experts per GPU; 32 total H20 GPUs across 4 nodes × 8 cards.
- Theoretical throughput estimate before Tencent data: 35,000–45,000 tokens/s (optimistic), reachable through MoE efficiency factor ≥85% and CUDA Graph + vLLM engine tuning.
- Per-GPU VRAM: 83.9–92.3 GB; model requires 1,342 GB BFloat16 total vs. 3,072 GB total cluster VRAM — fits comfortably.
- H20 GPU compute: 148 TFLOPS BF16; MLA attention layer FLOPs ≈ 0.12T/layer — compute deemed sufficient for 50,000 tokens/s.
- TTFT P50 projected at 211–374 ms for 512-token inputs — far below the 0.8 s target.
- Network: NVLink intra-node (900 GB/s), ROCEv2 + 25G inter-node; network optimization expected to yield 5–8% throughput gain.
