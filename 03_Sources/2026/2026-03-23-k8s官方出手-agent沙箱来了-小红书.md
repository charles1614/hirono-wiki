# [2026-03-23] K8s Official: Agent Sandbox Arrives

## TL;DR

Kubernetes SIG Apps has released Agent Sandbox, a Kubernetes-native CRD purpose-built for AI agent workloads. The design addresses the mismatch between traditional StatefulSet/PVC/Service approaches and agent characteristics (stateful, long-running, singleton, bursty, untrusted-code execution).

## Key claims

- AI agents differ fundamentally from microservices: stateful and long-running (hours to days), singleton (not horizontally scalable), bursty activity with extended idle periods, and requiring secure isolation for untrusted code execution.
- Agent Sandbox CRD provides four capabilities: (1) security isolation via gVisor/Kata dual-engine kernel-level sandboxing; (2) smart lifecycle management — idle-to-zero scaling with state-preserving wake; (3) stable identity discovery via per-agent hostname for multi-agent communication; (4) WarmPool pre-warming to eliminate cold-start latency.
- A single `SandboxClaim` resource allocates a pre-warmed sandbox environment immediately.
- Traditional K8s patterns (StatefulSet + PVC + Service) impose excessive operational complexity for agent workloads.
