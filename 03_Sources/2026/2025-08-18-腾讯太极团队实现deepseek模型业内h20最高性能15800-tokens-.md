# [2025-07-11] 腾讯太极团队实现 DeepSeek 模型业内 H20 最高性能 15,800+ tokens/s

## TL;DR

Tencent Taiji Angel-HCF inference team describes how they achieved the industry-record 15,800+ tokens/s on 16×H20-96G GPUs for DeepSeek-V3/R1, combining PD disaggregation, large Expert Parallelism, multi-layer MTP, w4a8c8 quantization, and Hopper-specific optimizations.

## Key claims

- Core result: 16-card H20 cluster, 50 ms TPOP limit, QPM=212, **15,800+ tokens/s** — industry's highest H20 throughput for DeepSeek as of publication.
- [[Inference Disaggregation]] (PD separation): Prefill uses large TP + small EP (compute-bound); Decode uses DP + large EP (memory-bound). KV Cache transferred asynchronously via RDMA, overlapping with forward step computation so GPU utilization stays high.
- Layerwise KV Cache transfer: for ISL > 8K, transfer layer-by-layer during forward pass to overlap latency; for ISL < 8K, bulk transfer with small-packet merging.
- EP optimization via TRMT communication library (Tencent Network team): pre-optimization EP communication overhead was 40%+ of timeline; after optimization, reduced by 60%.
- Expert load balancing with EPLB algorithm + dynamic load balancing + redundant experts: imbalance ratio reduced to 1.2–1.5 across ranks.
- Multi-layer [[MTP]]: trained 5 independent MTP layers; self-trained independent MTP2 improves over open-source MTP2 by 8.8%, MTP3 by 9.0%. Token acceptance rate reached ~0.7 per layer using "Typical Sampling" variant with dynamic temperature.
- DP parallel for Decode: borrowing from training frameworks, achieves 50%+ single-node throughput gain while meeting SLO.
- Next step target: 20,000+ tokens/s via TBO, full-global KV cache, and further DeepEP communication optimization.
