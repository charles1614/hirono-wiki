# [2025-08-01] 抖音全新推荐大模型RankMixer，参数翻70倍，推理成本不涨

## TL;DR

[[ByteDance]]'s algorithm team presents [[RankMixer]], a new recommendation ranking model that scales the Douyin (TikTok) feed ranking model from 16M to 1B dense parameters (~70×) while holding inference latency constant, by aligning model architecture to GPU compute patterns via TokenMixing and per-token SparseMoE.

## Key claims

- Baseline Dense-16M model had MFU in the low single digits (~4%) and SM Activity ~30%; [[RankMixer]]-1B raises MFU ~10× to 40%+ and SM Activity to 80%, reframing the problem from memory-bound to compute-bound.
- **Automatic Feature Tokenization** groups features by semantic prior, concatenates within groups, then uniformly splits into fixed-dimension tokens — avoiding per-feature token fragmentation and enabling parallel hardware utilization.
- **TokenMixing**: parameter-free cross-feature interaction that splits each token into H sub-vectors and concatenates corresponding slices across tokens. No learnable weights → no O(n²) attention map; avoids memory bandwidth pressure from Self-Attention's H × n² weight matrices. Empirically outperforms Self-Attention on recommendation data where feature semantics are highly heterogeneous.
- **Per-Token SparseMoE**: each token gets its own [[MoE]] FFN with sparse gating (ReLU routing adapts activated-expert count to token information density). DTSI ("Dense Training, Sparse Inference") keeps all experts active during training for uniform expert utilization, then applies sparse routing at inference.
- Latency cost decomposition: parameter count ↑ 70×, FLOPs ↑ only 20× (3.6× FLOPs-per-parameter reduction via design), then MFU ↑ 10× shrinks latency to 2× baseline, with half-precision and graph-level optimizations eliminating the residual gap.
- Online A/B test on Douyin main feed: 0.3% lift in LT30 (DAU proxy) and 1%+ in daily session duration; gains larger for low-active users, demonstrating improved generalization.
- As of paper publication (arXiv 2507.15551), deployed across dozens of ByteDance products including Douyin feed ranking and e-commerce ads.

## What this changes

Demonstrates that recommendation ranking models have a viable GPU-aligned scaling path; the key insight is that architectural GPU-alignment (large matmuls, no memory-bound ops) matters more than raw parameter count for practical scalability.
