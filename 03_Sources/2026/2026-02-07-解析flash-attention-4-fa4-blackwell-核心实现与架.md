# [2026-02-07] 解析Flash Attention 4（FA4）：Blackwell 核心实现与架构优化

## TL;DR
A deep technical analysis of Flash Attention 4 (FA4) for NVIDIA Blackwell GPUs, reverse-engineered from open source code. FA4's ~20% performance gain over cuDNN's best attention kernel comes primarily from a more complex async pipeline with warp specialization, plus two algorithmic innovations: a software polynomial approximation of exp replacing SFU calls, and a smarter online Softmax that reduces rescaling operations by 90%.

## Key claims
- FA4 shows ~20% performance improvement over cuDNN 9.11.0's attention kernel on Blackwell (tested with CTK 13.0); source: `flash_attn/cute/flash_fwd_sm100.py`.
- Five warp specialization roles: Load (global→shared memory via TMA), MMA (QK and PV matrix multiply-accumulate on Tensor Cores), Softmax (normalizes attention scores, tracks running max/sum), Correction (rescales historical outputs when scaling factor changes), Epilogue (shared→global memory write-out).
- New exp implementation: replaces SFU (`MUFU.EX2`) with a 3-term polynomial approximation via Horner's method (3 FMA ops) for the fractional part of 2^x; accuracy matches bf16 SFU output; used selectively to relieve SFU bottleneck in early iterations.
- Online Softmax improvement: rescaling is only triggered when the running max changes by an amount large enough to affect numerical stability — reduces correction operations by 90% vs. FA3's "rescale on every new max" policy.
- Core PTX instruction is `tcgen05.mma.cta_group::1` (5th-gen Tensor Core, Blackwell sm100), using single-CTA execution instead of 2SM/2CTA TPC-based multiply to simplify scheduling (at minor memory throughput cost).
- FA4 uses StaticPersistentTileScheduler: at most one CTA per SM, with fine-grained pipelining across tiles — reduces CTA launch overhead and enables cross-tile concurrency (Epilogue of tile N overlaps with Load/MMA of tile N+1).
- The article frames GPU programming's trajectory as toward tiled, warp-specialized pipelines; cites NVIDIA investments in CuTe DSL, [[CUTLASS]], and the forthcoming [[CUDA Tile]] as evidence; notes that the Triton team dropped Blackwell attention work in favor of the Gluon frontend.

## What this changes
FA4 establishes warp specialization with producer-consumer async pipelines as the new baseline for top-tier GPU kernel design on Blackwell, replacing FA3's "ping-pong" pipeline.
