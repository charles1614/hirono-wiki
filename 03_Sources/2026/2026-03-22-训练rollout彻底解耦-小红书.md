# [2026-03-20] ProRL Agent: Rollout-as-a-Service for RL Training of Multi-Turn LLM Agents

## TL;DR

ProRL Agent (arXiv:2603.18815) decouples multi-turn agent rollout execution from the RL training stack via a Rollout-as-a-Service architecture, improving GPU utilization and horizontal scalability. An 8B model trained with ProRL gains ~2 points over SkyRL-v0 on software engineering, STEM, math, and code generation benchmarks.

## Key claims

- Existing [[RL Post-Training]] frameworks tightly couple I/O-intensive agent execution with GPU-intensive training, causing resource contention and impeding large-scale [[RL Post-Training]] of multi-turn agents.
- ProRL Agent introduces a three-layer architecture: scalable sandbox environments, an async three-stage server (queue / execute / return), and a training-side client that communicates via HTTP — enabling token-level communication that avoids re-tokenization drift.
- Rootless deployment support (no privilege escalation) makes ProRL Agent compatible with HPC cluster environments where containers cannot run as root.
- On software engineering, STEM, math, and code generation tasks an 8B model trained with ProRL Agent improves nearly 2 points over SkyRL-v0; throughput scales linearly with the number of rollout servers.
