# [2026-03-18] NVIDIA GTC2026 详细解读和分析

## TL;DR

Detailed hardware-focused breakdown of NVIDIA GTC 2026 announcements, covering the GPU roadmap from Volta through Rubin and Feynman generations, GPU data processing libraries (cuDF, cuVS), inference-era demand trends, and new system-level tray/rack hardware.

## Key claims

- GB NVL72 + NVFP4 delivers 50× token throughput improvement or 35× token cost reduction vs H200 NVL8 at same power — based on DeepSeek-R1-0528 FP4, 1K/1K input/output tokens; B300 vs H100 FP8 is only ~2× at same power, so the 35–50× figure relies heavily on larger NVLink domain and higher concurrency headroom from bigger VRAM.
- [[NVIDIA]] hardware generational progression: [[Volta]] (V100 + NVLink-2), [[Ampere]] (A100 + NVLink-3 + CX-6 200Gb/s), [[Hopper]] (H100 + NVLink-4 + CX-7 400Gb/s), [[Blackwell]] (B200/B300 + NVLink-5 + CX-8 800Gb/s IB / 400Gb/s Eth), Rubin (CX-9 + NVLink-6 + BlueField-4), Feynman (NVLink-8 CPO + CX-10 1600Gb/s + Spectrum-7 204T).
- Vera Rubin vs X86+Hopper at same 1GW AI factory: 13.3× compute, 50× memory bandwidth (2EB/s vs 100EB/s), 350× token throughput (2M vs 700M TPS).
- GPU demand grew from $0.5T (GTC 2025 estimate) to $1T by GTC 2026; growth driven by inference era — model scale, context length, token count each grew ~10× from ChatGPT → Reasoning (o1) → Agentic (Claude Code) era.
- cuDF (CUDA DataFrame, RAPIDS) enables GPU-accelerated pandas-like operations; IBM case: data processing from 15 min → 3 min; Google Cloud A/B platform: resources from 45K CPU → 1K GPU, 76% cost reduction; Dell uses cuVS + cuDF for hour-to-minute enterprise data processing at 3× CPU speed.
- [[NVLink]] Switch Spine (9 NVSwitch trays) supports 72 ports at 1.8TB/s each = 130TB/s switching bandwidth in the NVL72 form factor.
