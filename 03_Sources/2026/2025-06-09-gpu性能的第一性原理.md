# [2025-05-31] GPU性能的第一性原理

## TL;DR

本文从计算密度（FLOPS/Data rate）出发梳理GPU性能的两大核心挑战：吞吐瓶颈（访存带宽增长有限）与指令访存效率低（单条指令可用带宽/总带宽比率低），并分别给出优化思路和架构实现案例。

## Key claims

- GPU性能的第一性原理：吞吐瓶颈长期是访存带宽，无突破性技术；效率取决于设备体系架构（能效比更高的指令、更高效的指令pipeline、更好的缓存命中效率）。
- 计算密度=FLOPS/Data rate，每代GPU都在下降，因为内存带宽增长速度远慢于算力增长；GPU吞吐高但计算密度低于CPU。
- 挑战1（计算强度）：矩阵张量（tensor）类计算才能少依赖内存带宽；卷积/线性计算强依赖——这是Transformer成功的原因之一（充分发挥GPU算力）。
- 挑战2（指令访存效率）：单条指令可使用的最大内存带宽/总内存带宽比率低。优化思路：①超量订阅线程（远超实际需要）通过流水线掩盖访存延时；②多级缓存增加有效带宽。
- 吞吐优化思路：低精度操作数、晶体管制程升级、稀疏矩阵加速、增大芯片体积。
- 架构案例：Sparse Tensor Core将稀疏矩阵分离为非零权重（数值）+非零索引，提升计算密度；分布式共享缓存（DCMEM）通过TMA组件实现SM2SM network，SM可访问其他SM的L1 cache，SM2SM延时180 cycles，比访问L2 cache低32%。
- GPU拓扑层级：GPU → cluster（一组SM，通过SM2SM network共享L1 cache）→ SM → sub-core → warp group → warp → thread。
- 作者观点（供讨论，非严格验证）：CPU长期将压倒GPU——GPU计算效率（非吞吐）极低，且优化已进入深水区；GPU演进方向为提升数据表达能力、稀疏矩阵加速、优化显存hierarchy、芯片封装、制程升级。
