# [2017-08-14] 高速信号编码之8B/10B

## TL;DR

Technical primer on [[8B10B Encoding]] for high-speed serial links, explaining the DC-balance motivation, the 5B/6B + 3B/4B split encoding structure, running disparity (RD) toggling mechanism, and a simulation validation using the D3.0 code word. Author introduces 8B/10B as the first in a two-part series also covering 64B/66B.

## Key claims

- AC-coupling capacitor in serial links has impedance Zc=1/2πfC; continuous 0s/1s create low-frequency patterns that cause large capacitive voltage drop, making bit identification impossible — encoding converts these to higher-frequency patterns.
- [[8B10B Encoding]] splits 8 bits into low 5 bits (ABCDE → 5B/6B encoded) and high 3 bits (FGH → 3B/4B encoded); resulting 10-bit codes contain only 5+5, 4+6, or 6+4 zero/one distributions.
- Running disparity (RD): if encoded word has more 1s than 0s → RD negative; more 0s → RD positive; encoder alternates RD polarity across consecutive words to maintain long-term DC balance.
- Each 8-bit input maps to two 10-bit codes (RD+ and RD−) via a lookup table; encoder tracks current disparity to choose the correct variant.
- Simulation of D3.0 code word confirms correct alternation between RD− and RD+ patterns.
- Efficiency: 80% (8 data bits per 10 transmitted bits); other schemes exist — 4B/5B (80%), 64B/66B (~97%), 128B/130B (~98%).
